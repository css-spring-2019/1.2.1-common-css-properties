Property Use Values
display Changes how an element is rendered. You can change inline tags to behave like full-width block tags for example. Options: block, inline, inline-block, flex, none (removed from document flow)
position Type of positioning. static is the default. Options: static, absolute, relative, fixed
top / right / bottom / left When using positioning, offset from edge. Example: top: -100px; left: 10%; offsets the box up 100 pixels from the top edge, and 10% of parent box width to the right of the left edge.
flex When using display: flex; on a parent element, this property specifies how many flex units the child element should consume. An number that's a fraction of the total number of flex children: .parent {display: flex;} .child {flex: 1;}
columns Automatically divides up a box and flows inline elements into columns. Value should be number of columns.
overflow Changes how content that is larger than specified width and height behaves. Options: visible (breaks out), hidden, auto (scrollbars)
float When using display: block; you can specify how that block element interacts with other block elements. You can float an image to the right of inline text for example. Options: left or right
z-index When using absolute, fixed, or relative positioning, you can specify the layer order of elements to ensure they display on top of the correct element. Value can be a negative number or a positive number based on other z-indexed layers.
